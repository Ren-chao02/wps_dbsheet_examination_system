## 简介
本文件基于 Prisma Schema 对考试系统的数据库表结构进行系统化梳理，逐表说明字段定义、数据类型、约束条件与索引策略；解释主键、外键、唯一性与非空设计考量；阐述 JSONB 字段的使用场景与数据格式；给出 SQL 创建语句与字段级注释模板；并提供命名规范、类型选择原则与性能优化建议，帮助数据库管理员建立完整的表结构参考手册。

## 详细表结构定义

### 用户表 users
- 字段与类型
  - id: 主键，UUID，默认自动生成
  - username: 唯一，长度上限 64
  - passwordHash: 密码哈希，长度上限 255
  - realName: 真实姓名，长度上限 128
  - role: 角色枚举（admin/teacher/student）
  - email: 邮箱，长度上限 255
  - avatarUrl: 头像地址，长度上限 512
  - createdAt/updatedAt: 时间戳，默认值与更新时间自动维护
- 约束与索引
  - 主键: id
  - 唯一: username
  - 非空: username, role, passwordHash
- 关系
  - 多对多: 作为创建者关联题目与考试
  - 一对多: 答题、会话、被评卷的提交
- 字段注释建议
  - id: 用户唯一标识
  - username: 登录用户名
  - passwordHash: 加盐哈希后的密码
  - realName: 显示名称
  - role: 用户角色
  - email/avatarUrl: 联系方式与头像
  - createdAt/updatedAt: 记录创建与更新时间

### 题目表 questions
- 字段与类型
  - id: 主键，UUID

  - title: 标题，长度上限 512
  - description: 描述，文本

- 约束与索引
  - 主键: id

  - 非空: title, type, difficulty, score, status
- 关系
  - 多对多: 与考试通过中间表关联
  - 一对多: 与答题明细关联
- 字段注释建议
  - id/title/description: 题目基本信息
  - type/difficulty/score/status: 题型、难度、分值与状态
  - answerRules: 判题规则（JSON 结构）

## 索引与约束策略
- 主键
  - 所有表均采用 UUID 主键，提升分布式安全性与可移植性。
- 外键
  - 明确声明外键关系，确保参照完整性；部分中间表使用级联删除保证数据一致性。
- 唯一性
  - 用户名唯一；考试题目与答题明细采用联合唯一，防止重复录入。
- 非空与默认值
  - 关键业务字段（如用户名、角色、题目标题、类型、难度、分数、状态等）均设置非空或合理默认值。
- 枚举
  - 使用 Prisma 枚举类型统一管理状态与类型，减少脏数据风险。
- JSONB
  - answer_rules、settings、answer_json、expected/actual 等字段采用 JSONB，便于灵活存储规则与动态结构。

## JSONB 字段使用规范
- 使用场景
  - answer_rules: 存储判题规则数组，支持不同题型的灵活判定逻辑。
  - settings: 存储考试配置（如是否允许回看、倒计时策略等）。
  - answer_json: 存储学生作答的原始结构化数据。
  - expected/actual: 存储期望与实际值，便于对比与审计。
- 数据格式建议
  - 统一使用对象或数组结构，字段命名清晰，避免深层嵌套。
  - 为关键字段提供注释与版本号，便于后续演进。
- 查询与索引
  - 如需高频查询特定键，可考虑引入生成列或 GIN 索引（PostgreSQL）以提升检索效率。

## SQL 创建语句与注释
以下为各表的 SQL 创建模板（字段类型映射至 PostgreSQL），并附带字段级注释说明。请根据实际部署环境调整序列/默认值与索引策略。

- users
  - 字段: id(uuid, 主键), username(varchar, 唯一), password_hash(varchar), real_name(varchar), role(enum), email(varchar), avatar_url(varchar), created_at(timestamptz), updated_at(timestamptz)
  - 约束: PRIMARY KEY(id), UNIQUE(username), NOT NULL(username, role, password_hash)
  - 注释: 用户唯一标识、登录名、密码哈希、真实姓名、角色、邮箱、头像URL、创建时间、更新时间
- question_categories
  - 字段: id(uuid, 主键), name(varchar), parent_id(uuid), sort_order(int), created_at(timestamptz)
  - 约束: PRIMARY KEY(id), NOT NULL(name), FOREIGN KEY(parent_id) REFERENCES question_categories(id)
  - 注释: 分类ID、名称、父分类、排序权重、创建时间
- questions
  - 字段: id(uuid, 主键), category_id(uuid), title(varchar), description(text), type(enum), difficulty(enum), score(int), answer_rules(jsonb), hints(text), tags(varchar[]), status(enum), created_by(uuid), created_at(timestamptz), updated_at(timestamptz)
  - 约束: PRIMARY KEY(id), NOT NULL(title, type, difficulty, score, status), FOREIGN KEY(category_id) REFERENCES question_categories(id), FOREIGN KEY(created_by) REFERENCES users(id)
  - 注释: 题目ID、分类ID、标题、描述、类型、难度、分数、判题规则、提示、标签、状态、创建者、创建时间、更新时间
- exams
  - 字段: id(uuid, 主键), title(varchar), description(text), mode(enum), duration_minutes(int), start_time(timestamptz), end_time(timestamptz), total_score(int), pass_score(int), status(enum), settings(jsonb), created_by(uuid), created_at(timestamptz), updated_at(timestamptz)
  - 约束: PRIMARY KEY(id), NOT NULL(title, mode, total_score, status), FOREIGN KEY(created_by) REFERENCES users(id)
  - 注释: 考试ID、标题、描述、模式、时长、开始/结束时间、总分、及格分、状态、配置、创建者、创建时间、更新时间
- exam_questions
  - 字段: id(uuid, 主键), exam_id(uuid), question_id(uuid), sort_order(int), score_override(int)
  - 约束: PRIMARY KEY(id), UNIQUE(exam_id, question_id), FOREIGN KEY(exam_id) REFERENCES exams(id) ON DELETE CASCADE, FOREIGN KEY(question_id) REFERENCES questions(id)
  - 注释: 关联ID、考试ID、题目ID、排序、分数覆盖
- student_submissions
  - 字段: id(uuid, 主键), exam_id(uuid), student_id(uuid), table_space_id(varchar), status(enum), started_at(timestamptz), submitted_at(timestamptz), graded_at(timestamptz), total_score(int), grader_comment(text), graded_by(uuid), created_at(timestamptz)
  - 约束: PRIMARY KEY(id), UNIQUE(exam_id, student_id), FOREIGN KEY(exam_id) REFERENCES exams(id), FOREIGN KEY(student_id) REFERENCES users(id), FOREIGN KEY(graded_by) REFERENCES users(id)
  - 注释: 答题ID、考试ID、学生ID、表空间ID、状态、开始/提交/评卷时间、总分、评语、评卷人、创建时间
- submission_details
  - 字段: id(uuid, 主键), submission_id(uuid), question_id(uuid), answer_json(jsonb), score(int), is_correct(boolean), created_at(timestamptz)
  - 约束: PRIMARY KEY(id), UNIQUE(submission_id, question_id), FOREIGN KEY(submission_id) REFERENCES student_submissions(id) ON DELETE CASCADE, FOREIGN KEY(question_id) REFERENCES questions(id)

## 命名规范与数据类型选择原则
- 命名规范
  - 表名与字段名采用小写加下划线风格，保持一致的可读性。
  - 外键字段以“表名_id”或“表名_id”的形式命名，便于识别。
  - 唯一约束字段使用“表名_字段名 UK”或直接标注 @unique。
- 数据类型选择原则
  - 主键统一使用 UUID，提升跨服务与分布式安全性。
  - 文本类字段按长度上限选择 varchar(n) 或 text，避免过度占用存储。
  - 时间字段使用 timestamptz 保存时区信息，便于全球用户。
  - IP 地址使用 inet 类型，便于网络相关校验与查询。
  - 枚举使用 Prisma 枚举，集中管理取值范围。
  - 动态结构使用 jsonb，兼顾灵活性与查询能力。
- 索引策略
  - 唯一键与外键自动建立索引；对高频过滤字段（如 username、exam_id、student_id）建议补充索引。
  - JSONB 字段若存在热点键查询，可考虑生成列 + GIN 索引。

## 性能优化建议
- 索引优化
  - 为高频查询字段添加合适索引（如 exam_id、student_id、username）。
  - 对 JSONB 字段热点键建立表达式索引或生成列索引。
- 查询优化
  - 使用分页与投影字段，避免 SELECT *。
  - 合理使用 JOIN，避免 N+1 查询。
- 存储与备份
  - 定期清理历史会话与过期答题，控制表规模。
  - 对大字段（如 text、jsonb）进行冷热分离或归档。
- 连接与并发
  - 使用连接池与只读副本，降低主库压力。
  - 对高并发写入场景启用批量插入与事务合并。

## 故障排查指南
- 连接问题
  - 确认 DATABASE_URL 环境变量正确，数据库可达。
  - 检查 Docker Compose 中数据库容器状态与端口映射。
- 数据不一致
  - 检查外键约束与级联删除策略，确认中间表唯一性约束生效。
  - 核对枚举值是否符合定义范围。
- JSONB 查询异常
  - 确认 JSONB 结构与键名一致，必要时添加表达式索引。
- 常用诊断命令
  - 使用 Prisma CLI 执行数据库迁移与 Studio 查看数据。
  - 通过种子脚本快速恢复初始数据进行验证。
